# Issue Creation Checklist

Before submitting an issue, confirm:

- [ ] Title clearly describes the task
- [ ] One WHAT label (component)
- [ ] One WHY label (type)
- [ ] One WHEN label (priority)
- [ ] Exactly one assignee
- [ ] Acceptance criteria defined

---

## Example

Title:
[TASK] Implement ALU shift operations

Labels:
design-alu
feature
P0-critical

Assignee:
One engineer
